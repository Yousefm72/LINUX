What is DNS

DNS (Domain Name System) הוא מערכת שמתרגמת שמות דומיין (כמו www.google.com) לכתובות IP (כמו 142.250.185.46). ללא DNS היינו צריכים לזכור כתובות IP לכל אתר.

DNS פועל בהיררכיה: root servers → TLD (.com, .org) → authoritative servers לדומיין. רשומות נפוצות: A (שם→IP), AAAA (IPv6), MX (מייל), CNAME (כינוי), NS (שרת DNS).

A record מזהה host לשם → כתובת IPv4.
MX record שרתי דוא"ל לדומיין.
CNAME כינוי (alias) – מצביע לשם אחר.

Configuring a DNS Client

לקוח DNS – המכונה שואלת שרתי DNS לתרגום שמות. ההגדרות נמצאות ב-/etc/resolv.conf או נשלטות על ידי NetworkManager/systemd-resolved.

/etc/resolv.conf קובץ הגדרות DNS – nameserver 8.8.8.8.
nameserver 8.8.8.8 שורת resolv.conf – מגדירה שרת DNS ראשי.
nameserver 1.1.1.1 Cloudflare DNS כחלופה.
/etc/systemd/resolved.conf הגדרות systemd-resolved (חלק מההפצות).
resolvectl status מציג את הגדרות ה-DNS הנוכחיות.

Using the nslookup Command

nslookup – כלי שאילתות DNS אינטראקטיבי או משורת פקודה. בודק תרגום שם→IP ולהיפך.

nslookup google.com שואל כתובת IP עבור google.com.
nslookup 8.8.8.8 שואל Reverse DNS – שם עבור כתובת IP.
nslookup -type=MX example.com שואל רשומות MX (שרתי מייל).
nslookup google.com 8.8.8.8 שואל ישירות משרת DNS ספציפי.

Using the dig Command and Other Tools

dig (Domain Information Groper) – כלי מתקדם לשאילתות DNS. מחזיר מידע מפורט. host – פשוט יותר. whois – מידע על רישום דומיין.

dig google.com שאילתת A – מציג תשובה מפורטת כולל TTL, authority.
dig google.com +short תשובה קצרה – רק כתובות IP.
dig MX example.com שאילתת MX.
dig @8.8.8.8 google.com שולח שאילה ישירות לשרת 8.8.8.8.
host google.com פקודה פשוטה – שם→IP.
whois example.com מידע על רישום הדומיין.

Administering a DNS Server

שרת DNS – BIND (named) הוא הנפוץ ב-Linux. מנהל zones, רשומות, ו-caching. קבצי הגדרה: /etc/bind/named.conf (Debian), /etc/named.conf (RHEL).

named תהליך BIND – שרת DNS.
systemctl status named בודק סטטוס שרת BIND.
/etc/bind/named.conf.local הגדרות zones מקומיות (Debian).
/var/named/ קבצי zone (רשומות A, MX, וכו') – RHEL.
named-checkconf בודק תחביר קובץ named.conf.
named-checkzone example.com /path/to/zone בודק תחביר קובץ zone.
rndc reload טוען מחדש zones בלי לאתחל את named.

Creating DNS Server Copies

עותקי שרת DNS (replicas/secondaries) – להעברת עומס ו-availability. Primary שולח zone transfers ל-Secondaries. הגדרת allow-transfer ו-slave ב-named.conf.

allow-transfer { 192.168.1.2; }; ב-named.conf – מאפשר zone transfer ל-secondary.
type slave; ב-zone block – secondary מקבל העתקה מ-primary.
masters { 192.168.1.1; }; מגדיר את כתובת ה-primary עבור secondary.
notify yes; primary יודיע ל-secondaries על עדכונים.

סיכום ותרגול בעברית עם דוגמאות

DNS: מתרגם שמות דומיין לכתובות IP. רשומות A, MX, CNAME, NS.

לקוח: /etc/resolv.conf, nameserver 8.8.8.8. resolvectl status.

פקודות בדיקה:

  • nslookup google.com – שאילתת DNS
  • dig google.com +short – תשובה קצרה
  • dig MX example.com – רשומות MX

תרגיל 1 – בדיקת DNS:

  • dig www.wikipedia.org +short
  • nslookup -type=NS google.com

תרגיל 2 – שרת BIND:

  • named-checkconf – בדיקת config
  • rndc reload – רענון zones
