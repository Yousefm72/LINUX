Samba Service Overview

Samba מיישם את פרוטוקול SMB/CIFS – הפרוטוקול שמשתמשים Windows לשיתוף קבצים ולהדפות. מאפשר ל-Linux לשתף תיקיות ולהתחבר ל-Windows networks, לשרת כמשאב רשת ב-domain, ולהירשם ל-Active Directory.

רכיבים עיקריים: smbd (שיתוף קבצים והדפות), nmbd (NetBIOS name resolution), winbindd (אימות מול Windows domain). קובץ ההגדרה: /etc/samba/smb.conf.

smbd שירות שיתוף קבצים והדפות (SMB).
nmbd שירות NetBIOS – שמות מכונות ברשת Windows.
/etc/samba/smb.conf קובץ ההגדרות הראשי של Samba.
smbclient -L //server מציג shares זמינים על שרת Samba.
testparm בודק תחביר smb.conf.

What is SSSD

SSSD (System Security Services Daemon) מספק גישה מרכזית לזהות ואימות – LDAP, Active Directory, Kerberos. מטפל ב-caching של credentials ומידע משתמשים/קבוצות, ומחליף את שילוב ה-ldap/nss/pam הישן.

SSSD מאפשר ל-Linux להצטרף ל-domain, לאמת משתמשים מול AD, ולגשת למשאבים משותפים. קובץ ההגדרה: /etc/sssd/sssd.conf.

sssd הדמון – מחבר ל-LDAP/AD ומגיש נתונים ל-NSS/PAM.
/etc/sssd/sssd.conf קובץ ההגדרות – domain, ldap_uri, ad_server.
realm join example.com מצרף את המכונה ל-Active Directory domain (עם realmd).
id user@domain בודק אם משתמש domain מזוהה (uid, gid, groups).

LDAP Overview

LDAP (Lightweight Directory Access Protocol) – פרוטוקול למאגרי מידע היררכיים (directory). משמש לאחסון משתמשים, קבוצות, מכשירים. Active Directory מבוסס על LDAP (עם הרחבות).

מבנה: DN (Distinguished Name), OU (Organizational Unit), CN (Common Name). LDAP רצה על פורט 389 (לא מוצפן) או 636 (LDAPS). OpenLDAP – מימוש קוד פתוח.

ldapsearch -x -b "dc=example,dc=com" שאילתת LDAP – מציג רשומות במאגר.
ldapadd מוסיף רשומות ל-LDAP.
port 389 / 636 LDAP ו-LDAPS (מוצפן).
389-ds / slapd שרת OpenLDAP – 389 Directory Server או slapd.

Connecting a Linux Server to Active Directory

חיבור Linux ל-Active Directory – המשתמשים יכולים להתחבר עם credentials של domain. כלים: realmd + sssd, או winbind (Samba) + Kerberos.

realm discover example.com מגלה את ה-domain ומציג מידע על AD.
realm join example.com -U admin מצרף את המכונה ל-domain. יוצר חשבון ב-AD.
realm leave מנתק את המכונה מה-domain.
kinit user@DOMAIN מקבל ticket Kerberos – אימות ל-domain.
systemctl restart sssd מרענן SSSD אחרי שינויים.

דרישות: DNS מכוון ל-AD, שעון מסונכרן (NTP), connectivity ל-domain controller.

סיכום ותרגול בעברית עם דוגמאות

Samba: שיתוף קבצים SMB/CIFS. smbd, nmbd. /etc/samba/smb.conf. testparm.

SSSD: אימות מרכזי ל-LDAP/AD. /etc/sssd/sssd.conf. realm join.

LDAP: פרוטוקול directory. ldapsearch, ldapadd. Active Directory מבוסס LDAP.

חיבור ל-AD: realmd + sssd. realm join, kinit. DNS ו-NTP חייבים להיות נכונים.

תרגיל 1 – Samba:

  • testparm – בדיקת smb.conf
  • smbclient -L //localhost -U user – רשימת shares

תרגיל 2 – Active Directory:

  • realm discover ad.example.com
  • realm join ad.example.com -U Administrator
  • id user@ad.example.com – בדיקת משתמש domain
