The process architecture

תהליך (process) הוא מופע של תוכנה שרצה. לכל תהליך יש PID (Process ID) ייחודי, PPID (Parent Process ID), מצב (running, sleeping, zombie), ומשאבים (זיכרון, file descriptors).

במערכות Unix-like, תהליך init/systemd הוא תהליך האב הראשון (PID 1). כל התהליכים נוצרים ב-fork ו-exec. תהליכים יכולים ליצור תהליכי ילד (child processes) ולהיות חלק מעץ תהליכים.

PID מזהה תהליך ייחודי (Process ID).
PPID מזהה תהליך האב (Parent Process ID).
/proc/ מערכת קבצים וירטואלית עם מידע על תהליכים (Linux).
fork() / exec() מנגנון יצירת תהליכים – שכפול ואז החלפת תמונה.
Owner המשתמש שהפעיל את התהליך (UID). קובע הרשאות גישה למשאבים.
GROUP הקבוצה של התהליך (GID). משפיעה על הרשאות קבוצתיות.
PRIORITY עדיפות התהליך – קובעת כמה זמן CPU יוקצה (nice value, 0 עד 39).
Scheduling Class מחלקת תזמון – איך ה-kernel מקצה זמן CPU (TS, RT, FF, SYS וכו').

Handling processes on Linux

ניהול תהליכים ב-Linux מתבצע עם פקודות כמו ps, top, kill, pgrep, pkill.

ps מציג תהליכים. דוגמאות: ps aux, ps -ef
top מציג תהליכים בזמן אמת עם עדכון דינמי.
htop ממשק משופר ל-top עם ניווט מקלדת.
pgrep pattern מחפש תהליכים לפי שם ומחזיר PIDs.
pkill pattern שולח סיגנל לתהליכים התואמים (ברירת מחדל: SIGTERM).
kill PID שולח SIGTERM (15) – בקשה לעצירה עדינה.
kill -9 PID שולח SIGKILL (9) – עצירה מיידית.
killall processname שולח סיגנל לכל התהליכים בשם נתון.
nice / renice משנה עדיפות (priority) של תהליך.
nohup command מריץ פקודה שתמשיך גם אחרי ניתוק הטרמינל.

Handling processes on Solaris

Solaris (Oracle) משתמש בכלים דומים אך עם הבדלים. prstat דומה ל-top, pgrep ו-pkill קיימים, ו-ps עם אופציות שונות.

prstat מציג תהליכים בזמן אמת (מקביל ל-top ב-Linux).
prstat -a מציג תהליכים לפי משתמש.
ps -ef מציג את כל התהליכים (פורמט מלא).
ps -e -o pid,ppid,comm מציג PID, PPID ושם פקודה.
pgrep -l pattern מחפש תהליכים ומציג PID ושם.
pkill pattern שולח SIGTERM לתהליכים התואמים.
kill PID שולח סיגנל לתהליך (SIGTERM).
kill -9 PID SIGKILL – עצירה כפויה.
preap PID מנקה תהליכי zombie (reap) ב-Solaris.

סיכום בעברית עם דוגמאות

ארכיטקטורת תהליכים: לכל תהליך PID ו-PPID. init/systemd הוא PID 1. תהליכים נוצרים ב-fork/exec.

Linux – פקודות עיקריות:

  • ps aux – הצגת כל התהליכים
  • top – מעקב בזמן אמת
  • kill 1234 – עצירה עדינה (SIGTERM)
  • kill -9 1234 – עצירה כפויה (SIGKILL)
  • pgrep nginx – חיפוש PID של nginx

Solaris – פקודות עיקריות:

  • prstat – מעקב תהליכים (מקביל ל-top)
  • ps -ef – רשימת תהליכים
  • pkill processname – שליחת סיגנל לפי שם
  • preap PID – ניקוי zombie

הבדל מרכזי: Solaris משתמש ב-prstat במקום top, ו-preap לניקוי תהליכי zombie.
