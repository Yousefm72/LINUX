User Management

Users and groups configuration files

מידע על משתמשים וקבוצות נשמר בקבצי טקסט. עריכת הקבצים ישירות מסוכנת; עדיף להשתמש בפקודות הניהול.

/etc/passwd – רשימת משתמשים. כל שורה בפורמט:

username:x:UID:GID:comment:home_dir:shell
username
שם המשתמש להתחברות. דוגמה: root, david
x
מציין שהסיסמה נשמרת ב-shadow (תמיד x). דוגמה: x
UID
מזהה משתמש ייחודי. 0=root, 1-999=משתמשי מערכת, 1000+=משתמשים רגילים. דוגמה: 0, 1000
GID
מזהה קבוצה ראשית. דוגמה: 0, 100
comment
שדה הערה (שם מלא או תיאור). דוגמה: root, David Cohen
home_dir
נתיב תיקיית הבית. דוגמה: /root, /home/david
shell
נתיב ה-shell ברירת המחדל. דוגמה: /bin/bash, /usr/sbin/nologin

דוגמה מלאה: root:x:0:0:root:/root:/bin/bash

/etc/group – קבוצות. כל שורה בפורמט:

groupname:x:GID:member1,member2,...
groupname
שם הקבוצה. דוגמה: users, sudo
x
מציין שסיסמת הקבוצה ב-gshadow (תמיד x). דוגמה: x
GID
מזהה קבוצה ייחודי. דוגמה: 100, 27
members
רשימת משתמשים בקבוצה, מופרדים בפסיק. דוגמה: alice,bob, david

דוגמה מלאה: users:x:100:alice,bob,david

/etc/shadow – סיסמאות מוצפנות (נגיש רק ל-root). פורמט:

username:encrypted_password:last_changed:min:max:warn:inactive:expire:reserved
username
שם המשתמש (תואם ל-passwd). דוגמה: root
encrypted_password
סיסמה מוצפנת. $6$=SHA-512, !!=חשבון נעול. דוגמה: $6$xyz...abc
last_changed
יום שינוי אחרון (מ-1.1.1970). דוגמה: 19000
min
ימים מינימליים בין החלפות סיסמה. 0=ללא הגבלה. דוגמה: 0
max
ימים עד תפוגת סיסמה. 99999=ללא תפוגה. דוגמה: 99999
warn
ימים לפני תפוגה להצגת אזהרה. דוגמה: 7
inactive
ימים לאחר תפוגה עד נעילת חשבון. דוגמה: -1 (ללא)
expire
תאריך תפוגת חשבון (יום). דוגמה: -1 (ללא)
reserved
שדה שמור לשימוש עתידי. דוגמה: ריק

דוגמה מלאה: root:$6$xyz...:19000:0:99999:7:::

/etc/gshadow – סיסמאות קבוצות. פורמט:

groupname:encrypted_password:admins:members
groupname
שם הקבוצה. דוגמה: sudo
encrypted_password
סיסמת קבוצה (לשימוש ב-newgrp). ! או !!=ללא סיסמה. דוגמה: !
admins
משתמשים שיכולים לנהל את הקבוצה. דוגמה: root,david
members
חברי הקבוצה (תואם ל-/etc/group). דוגמה: alice,bob

דוגמה מלאה: sudo:!::david,alice

Handling groups

groupadd groupname יוצר קבוצה חדשה. דוגמה: groupadd developers

שדות ואופציות:

-g GID
מגדיר מזהה קבוצה ספציפי. דוגמה: groupadd -g 1500 developers
-r
יוצר קבוצת מערכת (GID נמוך). דוגמה: groupadd -r sysgroup
-o
מאפשר GID כפול. דוגמה: groupadd -o -g 100 newgroup
groupmod [options] groupname משנה מאפייני קבוצה קיימת.
-n newname
משנה את שם הקבוצה. דוגמה: groupmod -n devs developers
-g GID
משנה את מזהה הקבוצה. דוגמה: groupmod -g 1600 developers
groupdel groupname מוחק קבוצה. הקבוצה חייבת להיות ריקה. דוגמה: groupdel oldgroup
gpasswd -a user group מוסיף משתמש לקבוצה. דוגמה: gpasswd -a david developers
gpasswd -d user group מסיר משתמש מקבוצה. דוגמה: gpasswd -d david developers
gpasswd -A user group מגדיר מנהל קבוצה (יכול להוסיף/להסיר חברים). דוגמה: gpasswd -A alice developers
gpasswd -r group מסיר סיסמה מקבוצה. דוגמה: gpasswd -r developers
newgrp groupname מחליף את הקבוצה הפעילה של המשתמש (להרצת הפקודה הנוכחית). דוגמה: newgrp developers
groups [username] מציג את הקבוצות של משתמש. בלי שם – המשתמש הנוכחי. דוגמה: groups david
getent group groupname מציג רשומת קבוצה מ-/etc/group. דוגמה: getent group developers

Handling users

useradd יוצר משתמש חדש; usermod משנה מאפיינים (שם, תיקייה, shell, קבוצות); userdel מוחק משתמש (אופציה -r מוחקת גם תיקיית הבית). passwd משנה סיסמה; chage מנהל תפוגת סיסמה. id מציג UID, GID וקבוצות של משתמש.

Login status

who מציג משתמשים מחוברים למערכת – שם משתמש, טרמינל, זמן כניסה.
who -a מציג מידע מלא על כל הכניסות הפעילות.
w מציג משתמשים מחוברים יחד עם הפקודה הפעילה שלהם וזמן idle.
w [username] מציג סטטוס כניסה עבור משתמש ספציפי.
last מציג היסטוריית כניסות מהקובץ /var/log/wtmp.
last -n 20 מציג 20 הכניסות האחרונות.
lastlog מציג את זמן הכניסה האחרון לכל משתמש במערכת.
users מציג רשימה קצרה של שמות המשתמשים המחוברים.
id מציג UID, GID וקבוצות של המשתמש הנוכחי.

The su command

su (substitute user) מאפשר להתחלף למשתמש אחר. דורש סיסמת היעד (root יכול להחליף בלי סיסמה).

su username נכנס כמשתמש אך שומר את סביבת המשתמש הנוכחי (PATH, תיקייה). דוגמה: su david
su - username כניסה מלאה (login shell) – טוען את סביבת המשתמש היעד. דוגמה: su - david
su - מעבר ל-root עם login shell. דוגמה: su -
su root מעבר ל-root בלי login shell. דוגמה: su root
su -c "command" username מריץ פקודה בודדת כמשתמש אחר וחוזר. דוגמה: su -c "whoami" david

שדות ואופציות:

- או -l או --login
כניסה מלאה – טוען סביבת משתמש (HOME, USER, SHELL). דוגמה: su -l david
-c command
מריץ פקודה בודדת. דוגמה: su -c "ls /root" root
-s shell
מגדיר shell ספציפי. דוגמה: su -s /bin/bash david
-m או -p או --preserve-environment
שומר את סביבת המשתמש הנוכחי (HOME, USER). דוגמה: su -m david
username
שם המשתמש היעד. ברירת מחדל: root. דוגמה: su alice

The sudo command

sudo מריץ פקודה כעל-משתמש (בדרך כלל root) בלי לצאת מהמשתמש הנוכחי. ההרשאות מוגדרות ב-/etc/sudoers. כל פעולה נרשמת – עדיף מ-su לאבטחה.

sudo command מריץ פקודה בודדת כ-root. דוגמה: sudo apt update
sudo -i פותח login shell של root (סביבה מלאה). דוגמה: sudo -i
sudo -s פותח shell של root אך שומר את תיקיית המשתמש הנוכחי. דוגמה: sudo -s
sudo -u user command מריץ פקודה כמשתמש אחר. דוגמה: sudo -u david whoami
sudo -l מציג את הפקודות שהמשתמש מורשה להריץ. דוגמה: sudo -l
sudo -v מרענן את תוקף הסיסמה (מאריך את זמן ההרשאה). דוגמה: sudo -v
sudo -k מבטל את תוקף הסיסמה – הפקודה הבאה תדרוש סיסמה מחדש. דוגמה: sudo -k
visudo עורך את קובץ /etc/sudoers בצורה מאובטחת (בדיקת תחביר). דוגמה: sudo visudo

שדות ואופציות:

-i או --login
login shell של root – טוען HOME=/root, USER=root. דוגמה: sudo -i
-s או --shell
מריץ shell של root אך שומר את HOME הנוכחי. דוגמה: sudo -s
-u user או --user=user
מריץ כמשתמש שצוין. דוגמה: sudo -u www-data cat /var/log/nginx/error.log
-l או --list
מציג הרשאות sudo של המשתמש. דוגמה: sudo -l
-v או --validate
מרענן timeout – מאריך את זמן ההרשאה (ברירת מחדל 15 דקות). דוגמה: sudo -v
-k או --reset-timestamp
מאפס את ה-timeout – הפקודה הבאה תבקש סיסמה. דוגמה: sudo -k
-b או --background
מריץ את הפקודה ברקע. דוגמה: sudo -b service nginx start
-n או --non-interactive
לא מבקש סיסמה – נכשל אם נדרשת הרשאה. דוגמה: sudo -n true
