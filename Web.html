Web Server Basics

שרת Web מגיש דפי HTML, קבצים סטטיים ואפליקציות דינמיות ללקוחות דרך HTTP/HTTPS. השרתים הנפוצים: Apache (httpd), Nginx, Caddy.

Apache – השרת הנפוץ ביותר, תומך ב-modules (PHP, SSL, proxy). Nginx – קל, מהיר, מתאים ל-reverse proxy. פורט 80 (HTTP), 443 (HTTPS).

httpd / apache2 תהליך Apache – RHEL משתמש ב-httpd, Debian ב-apache2.
nginx שרת Nginx – קל ומהיר.
DocumentRoot תיקיית השורש לתוכן האתר – בדרך כלל /var/www/html.
port 80 / 443 HTTP ו-HTTPS. דורש הרשאות או capabilities.

Server Administration - Apache

ניהול Apache – הקבצים העיקריים: /etc/httpd/conf/httpd.conf (RHEL) או /etc/apache2/apache2.conf (Debian). תיקיית sites: /etc/apache2/sites-available/.

systemctl start httpd מפעיל Apache (RHEL/Fedora).
systemctl start apache2 מפעיל Apache (Debian/Ubuntu).
apachectl configtest בודק תחביר קובץ הגדרות.
systemctl reload httpd טוען מחדש הגדרות בלי לנתק חיבורים.
/var/log/httpd/access_log לוג גישה – בקשות שהתקבלו.
/var/log/httpd/error_log לוג שגיאות.
a2enmod mod_ssl מפעיל מודול (Debian) – דוגמה: SSL.

Virtual Hosts

Virtual Hosts מאפשרים להריץ מספר אתרים על שרת אחד – לפי שם (name-based) או לפי IP. כל VirtualHost מגדיר DocumentRoot, ServerName, ואופציונלי ServerAlias.

Example VirtualHost (Apache):

<VirtualHost *:80>
    ServerName www.example.com
    DocumentRoot /var/www/example
    <Directory /var/www/example>
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
a2ensite example.conf מפעיל VirtualHost (Debian – יוצר קישור סימבולי).
a2dissite 000-default משבית אתר ברירת מחדל.
ServerName / DocumentRoot שם הדומיין ותיקיית התוכן.

Server Configuration - Reverse Proxy

Reverse Proxy מקבל בקשות מהלקוח ומעביר אותן לשרתים פנימיים (אפליקציה, API). מסתיר את המבנה הפנימי, מאפשר SSL termination, load balancing. Apache: mod_proxy; Nginx: proxy_pass.

Apache – mod_proxy:

ProxyPass /app http://localhost:8080/
ProxyPassReverse /app http://localhost:8080/

Nginx – proxy_pass:

location /app {
    proxy_pass http://localhost:8080/;
    proxy_set_header Host $host;
}
ProxyPass מעביר בקשות לשרת backend (Apache).
proxy_pass מעביר בקשות (Nginx).
a2enmod proxy proxy_http מפעיל מודולי proxy ב-Apache.

סיכום ותרגול בעברית עם דוגמאות

שרת Web: Apache, Nginx. פורט 80/443. DocumentRoot – תוכן האתר.

ניהול Apache:

  • systemctl start apache2 – הפעלה
  • apachectl configtest – בדיקת config

Virtual Hosts: ServerName, DocumentRoot. a2ensite / a2dissite.

Reverse Proxy: ProxyPass (Apache), proxy_pass (Nginx) – העברת בקשות ל-backend.

תרגיל 1 – VirtualHost:

  • צור קובץ ב-sites-available עם ServerName ו-DocumentRoot
  • a2ensite mysite.conf && systemctl reload apache2

תרגיל 2 – Reverse Proxy:

  • a2enmod proxy proxy_http
  • הוסף ProxyPass ו-ProxyPassReverse ל-VirtualHost
