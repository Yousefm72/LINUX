Linux Run Levels

רמות ריצה (Run Levels) מגדירות את מצב המערכת – אילו שירותים פעילים. במערכות SysV init יש רמות 0–6.

Run Level 0 – Halt (כיבוי המערכת)

Run Level 1 – Single-user mode (מצב משתמש יחיד, תיקון)

Run Level 2 – Multi-user, no NFS (רב-משתמש ללא רשת)

Run Level 3 – Full multi-user (רב-משתמש מלא, טקסט)

Run Level 4 – Unused (לא בשימוש)

Run Level 5 – Graphical (ממשק גרפי)

Run Level 6 – Reboot (הפעלה מחדש)

runlevel מציג את רמת הריצה הנוכחית והקודמת.
init 3 מעבר לרמת ריצה 3 (רב-משתמש).

SYS-V Services

שירותי SysV (System V) הם שיטת הניהול המסורתית. כל שירות הוא סקריפט ב-/etc/init.d/ עם פקודות start, stop, restart, status. קישורים סימבוליים ב-/etc/rcN.d/ (N = רמת ריצה) קובעים מתי השירות מופעל.

/etc/init.d/ תיקיית סקריפטי השירותים.
/etc/rc.d/rc3.d/ קישורים לשירותים ברמת ריצה 3 (S=start, K=kill).
chkconfig --list מציג את מצב השירותים בכל רמות הריצה (RHEL/Fedora).

Handling SYS-V Services

ניהול שירותי SysV מתבצע דרך הפקודה service או ישירות דרך הסקריפטים.

service servicename start מפעיל שירות.
service servicename stop עוצר שירות.
service servicename restart מפעיל מחדש שירות.
service servicename status מציג סטטוס שירות.
chkconfig servicename on מפעיל שירות באתחול (RHEL/Fedora).
chkconfig servicename off מבטל הפעלה אוטומטית.
update-rc.d servicename defaults מפעיל שירות באתחול (Debian/Ubuntu).

SYSTEMD Services

systemd הוא מנהל השירותים המודרני ב-Linux. הוא מחליף את init ומנהל יחידות (units) – שירותים, sockets, targets. קבצי יחידה נמצאים ב-/etc/systemd/system/ ו-/lib/systemd/system/.

.service סוג יחידה – שירות/דמון.
.target קבוצת יחידות (מקביל ל-run level).
/etc/systemd/system/ יחידות מותאמות מנהל.
/lib/systemd/system/ יחידות מהחבילות.

Handling SYSTEMD Services

ניהול שירותי systemd מתבצע דרך systemctl.

systemctl start servicename מפעיל שירות.
systemctl stop servicename עוצר שירות.
systemctl restart servicename מפעיל מחדש שירות.
systemctl reload servicename טוען מחדש הגדרות בלי לעצור.
systemctl status servicename מציג סטטוס מפורט.
systemctl enable servicename מפעיל שירות באתחול.
systemctl disable servicename מבטל הפעלה אוטומטית.
systemctl list-units --type=service מציג את כל השירותים הפעילים.
systemctl get-default מציג את ה-target ברירת המחדל.

Dependencies

שירותים יכולים להיות תלויים בשירותים אחרים. systemd מנהל תלויות אוטומטית – שירות לא יופעל לפני שהתלויות שלו פעילות. ב-SysV יש לטפל בסדר ההפעלה ידנית (מספרי S/K).

After=network.target שירות יופעל אחרי network.target.
Requires=sshd.service דורש ש-sshd פעיל.
systemctl list-dependencies servicename מציג את תלויות השירות.

Create Your Own Service

ליצירת שירות systemd, יוצרים קובץ .service ב-/etc/systemd/system/ ומריצים systemctl daemon-reload.

Example: /etc/systemd/system/myservice.service

[Unit]
Description=My Custom Service
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/myapp
Restart=on-failure

[Install]
WantedBy=multi-user.target
systemctl daemon-reload טוען מחדש את קבצי היחידות.
systemctl enable myservice מפעיל את השירות באתחול.
systemctl start myservice מפעיל את השירות כעת.

סיכום בעברית עם דוגמאות

רמות ריצה: 0=כיבוי, 1=משתמש יחיד, 3=רב-משתמש, 5=גרפי, 6=הפעלה מחדש.

SysV: שירותים ב-/etc/init.d/, ניהול עם service ו-chkconfig.

systemd: יחידות ב-/etc/systemd/system/, ניהול עם systemctl.

דוגמאות פקודות:

  • systemctl start nginx – מפעיל nginx
  • systemctl enable nginx – הפעלה אוטומטית באתחול
  • systemctl status nginx – מציג סטטוס
  • service apache2 restart – הפעלה מחדש (SysV)

יצירת שירות: קובץ .service עם [Unit], [Service], [Install], ואז daemon-reload ו-enable.
